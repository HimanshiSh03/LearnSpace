## **1. Fork the Repository**

1. Go to the [LearnSpace GitHub page](https://github.com/HimanshiSh03/LearnSpace).
2. Click the **Fork** button at the top-right corner.
3. This creates a copy of the repository under your GitHub account.

---

## **2. Clone Your Fork**

Clone your forked repository to your local machine:

```bash
git clone https://github.com/<your-username>/LearnSpace.git
cd LearnSpace
```

> Replace `<your-username>` with your GitHub username.

---

## **3. Branch Naming**

Create a descriptive branch for your work:

```bash
git checkout -b feature/short-description
git checkout -b bugfix/short-description
```

**Examples:**

* `feature/add-login-page`
* `bugfix/fix-task-api`

---

## **4. Commit Messages**

Write clear, concise commit messages using prefixes:

| Prefix   | Use For               |
| -------- | --------------------- |
| `feat:`  | New feature           |
| `fix:`   | Bug fixes             |
| `docs:`  | Documentation updates |
| `chore:` | Maintenance tasks     |

**Example:**

```
feat: add CONTRIBUTING.md to guide new contributors
```

---

## **5. Making a Pull Request (PR)**

1. Push your branch to your fork:

```bash
git push origin <branch-name>
```

2. Go to your fork on GitHub.
3. Click **Compare & Pull Request**.
4. Fill in a clear title and description of your changes.
5. Submit the PR.

---

## **6. Code Style Guidelines**

* Use **2 spaces** for indentation.
* Use **consistent naming and semicolons** in JS/Node projects.
* Follow **existing code style** in the repository.

---

## **7. Reporting Issues**

* Search existing issues before creating a new one.
* Provide details: steps to reproduce, expected vs actual behavior.
* Include screenshots or logs if helpful.

---
